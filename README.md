# Book Management API

## Описание проекта
Этот API предоставляет функциональность для управления книгами. С помощью этого API можно:
- Регистрировать пользователей.
- Авторизовываться и получать JWT токен.
- Создавать, обновлять, получать и удалять книги.
  
Приложение использует Flask для создания API, SQLAlchemy для работы с базой данных, и JWT для авторизации пользователей.

### Предметная область
Сущность **Book** (Книга) имеет следующие поля:
- `id`: уникальный идентификатор книги (integer)
- `title`: название книги (string)
- `author`: автор книги (string)
- `genre`: жанр книги (string)
- `year`: год выпуска книги (integer)
- `available`: доступность книги (boolean)

## Стек технологий
- **Backend**: Python, Flask, Flask-JWT-Extended
- **База данных**: PostgreSQL 17
- **Авторизация**: JWT
- **Миграции базы данных**: Flask-Migrate


## Установка и запуск

1. **Клонировать репозиторий**:
   ```bash
   git clone https://github.com/yourusername/book-management-api.git
````

2. **Установить зависимости**:

   ```bash
   pip install -r requirements.txt
   ```

3. **Запуск проекта**:

   ```bash
   flask run
   ```

   Приложение будет доступно по адресу: `http://localhost:5000`.

## API Документация

API поддерживает следующие маршруты:

### Авторизация

#### 1. **POST /auth/register**

* **Описание**: Регистрация нового пользователя.
* **Тело запроса**:

  ```json
  {
    "username": "new_user",
    "password": "password123"
  }
  ```
* **Ответ**:

  ```json
  {
    "msg": "Пользователь зарегистрирован",
    "user_id": 1
  }
  ```

#### 2. **POST /auth/login**

* **Описание**: Вход пользователя в систему. Возвращает JWT токен.
* **Тело запроса**:

  ```json
  {
    "username": "new_user",
    "password": "password123"
  }
  ```
* **Ответ**:

  ```json
  {
    "access_token": "jwt_token"
  }
  ```

#### 3. **POST /auth/logout**

* **Описание**: Выход пользователя из системы.
* **Ответ**:

  ```json
  {
    "msg": "Выход успешен"
  }
  ```

### CRUD операции с книгами

#### 4. **GET /books**

* **Описание**: Получить список всех книг.
* **Ответ**:

  ```json
  [
    {
        "title": "Властелин колец",
        "author": "Дж. К. Роулинг",
        "genre": "Фэнтези",
        "year": 1997,
        "available": true
    },
    {
        "title": "Гарри Поттер",
        "author": "Дж. Р. Р. Толкин",
        "genre": "Эпическое фэнтези",
        "year": 1937,
        "available": false
    }
  ]
  ```

#### 5. **GET /books/{id}**

* **Описание**: Получить книгу по ID.
* **Ответ (успешный)**:

  ```json
  {
    "title": "Властелин колец",
    "author": "Дж. Р. Р. Толкин",
    "genre": "Эпическое фэнтези",
    "year": 1937,
    "available": false
  }
  ```
* **Ответ (не найдено)**:

  ```json
  {
    "error": {
      "code": 404,
      "message": "Книга не найдена"
    }
  }
  ```

#### 6. **POST /books**

* **Описание**: Создать новую книгу.
* **Тело запроса**:

  ```json
  {
    "title": "Властелин колец",
    "author": "Дж. Р. Р. Толкин",
    "genre": "Эпическое фэнтези",
    "year": 1937,
    "available": false
  }
  ```
* **Ответ**:

  ```json
  {
    "book": {
        "title": "Властелин колец",
        "author": "Дж. Р. Р. Толкин",
        "genre": "Эпическое фэнтези",
        "year": 1937,
        "available": false
    },
    "msg": "Книга создана"
  }
  ```

#### 7. **PUT /books/{id}**

* **Описание**: Полное обновление книги.
* **Тело запроса**:

  ```json
  {
    "title": "Updated Book Title",
    "author": "Updated Author",
    "genre": "Non-Fiction",
    "year": 2022,
    "available": false
  }
  ```
* **Ответ**:

  ```json
  {
    "msg": "Книга обновлена"
  }
  ```

#### 8. **PATCH /books/{id}**

* **Описание**: Частичное обновление книги.
* **Тело запроса**:

  ```json
  {
    "available": false
  }
  ```
* **Ответ**:

  ```json
  {
    "msg": "Книга частично обновлена"
  }
  ```

#### 9. **DELETE /books/{id}**

* **Описание**: Удалить книгу по ID.
* **Ответ**:

  ```json
  {
    "msg": "Книга удалена"
  }
  ```

## Обработка ошибок

Ошибки возвращаются в формате JSON с соответствующими кодами:

* **400 Bad Request**: Неверный формат запроса или отсутствуют обязательные поля.
* **401 Unauthorized**: Ошибка авторизации (отсутствует токен или он неверен).
* **404 Not Found**: Ресурс не найден.
* **422 Unprocessable Entity**: Ошибка валидации данных.
* **500 Internal Server Error**: Внутренняя ошибка сервера.

Пример ошибки:

```json
{
  "error": {
    "code": 404,
    "message": "Книга не найдена"
  }
}
```
